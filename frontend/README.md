# frontend/ — SPA «Граф знаний R&D» (дизайнерский)

Одностраничное приложение (статический `index.html` + собранный `data.json`),
которое работает **без бэкенда и без ключей** прямо сейчас, и автоматически
подключается к REST API (`../api/server.py`), когда он запущен. Визуальный
стиль — тёмный графит + фирменный magenta/cyan-акцент Норникеля. Без CDN —
работает офлайн из коробки.

Создано как замена скучному Streamlit-фронтенду (`../frontend.py`) для демо:
полный контроль над разметкой/CSS/анимацией вместо streamlit-виджетов.

## Запуск

```bash
# 1. собрать данные (один раз, после каждого нового прогона ингеста)
python build_frontend_data.py
# → frontend/data.json (агрегат extraction_results_statyi_batch.jsonl)

# 2. поднять статику
python -m http.server -d frontend 8050
# → откройте http://localhost:8050
```

`data.json` уже собран из `extraction_results_statyi_batch.jsonl` (60
источников, 9750 сущностей, 4935 связей, 3483 числовых значения).

## Что внутри (по ТЗ хакатона)

**Поиск и навигация**
- Командная строка с семантическим поиском на естественном языке + 5
  примеров-чипов.
- Авто-разбор запроса в маркеры: числовые ограничения · география ·
  временной диапазон · сравнение · определение · причина-следствие —
  зеркалит `../routing/query_entity_extractor.py`.
- Бейдж маршрута (`GRAPH_ONLY` / `HYBRID` / `RAG_ONLY` / `NO_DATA`) —
  зеркалит `../routing/query_router.py`.
- Вкладка «Поиск и фильтры»: многоуровневая фильтрация по типу сущности,
  географии, типу связи, достоверности, числовому свойству + диапазону
  min/max — «сульфаты < 200 мг/л».
- Вкладка «Граф»: интерактивный force-directed граф (140 узлов, перетаскивание,
  зум, карточка сущности по клику) + технологические цепочки
  «материал → процесс → оборудование → результат».

**Аналитика и синтез ответов**
- Вкладка «Ответ»: структурированный ответ — найденные сущности, факты с
  цитатами, числовые ограничения, гео-разрез, источники. С API — синтез
  через LLM; без — локальная агрегация.
- Вкладка «Литературный обзор»: группировка источников по релевантности,
  консенсусные сущности (≥2 источников), зоны разногласий (`contradicts`),
  степень уверенности + число подтверждающих источников, рекомендации
  (похожие кейсы, эксперты, смежные темы).
- Вкладка «Пробелы и противоречия»: изолированные сущности, непокрытые
  пары «процесс × материал», отсутствие зарубежных источников, низко-
  уверенные факты + явные `contradicts`.

**Дашборды для руководителей**
- Вкладка «Дашборд»: KPI (сущности/связи/числа/эксперты), покрытие по
  направлениям (гидрометаллургия/пирометаллургия/экология/…), донат по
  типам сущностей, зоны риска (противоречия/мало данных), активность
  команд, типы связей.

**Управление доступом и безопасность**
- Ролевая модель в топбаре (исследователь/аналитик/руководитель/
  администратор/внешний партнёр) — меняет дефолтную вкладку и пишется в
  аудит.
- Вкладка «Аудит»: журнал запросов/просмотров/экспорта в localStorage,
  экспорт в JSON.

**Дополнительные пожелания**
- Мультиязычность: поиск матчит RU/EN сущности по 5-символьному префиксу
  токена (терпит морфологию: «обессоливания» ↔ «обессоливание»).
- Экспорт: Markdown, JSON-LD, PDF (через печать).
- Сравнительный анализ: вкладка «Сравнение» — таблица вариант А vs Б по
  численным параметрам (сид-сущности, связи, источники, противоречия) +
  качественные метрики (эффективность/затраты/холодный климат/экология).
- Подсветка пробелов и противоречий — в графе (красные пунктирные рёбра
  `contradicts`) и в отдельной вкладке.

## Связь с бэкендом

SPA сам определяет, есть ли API: пробует `GET /health` по URL из поля
топбара (по умолчанию `http://localhost:8080`). На связи — отправляет
`/query` и показывает LLM-синтез; офлайн — работает по `data.json`. Поле
URL и кнопка ↻ позволяют переключаться на лету.
